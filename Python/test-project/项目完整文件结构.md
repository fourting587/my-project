# ============================================================
# 关键文件说明
# ============================================================

## 1. 配置文件类

| 文件 | 作用 | 必须修改 |
|------|------|----------|
| `.env` | 数据库密码、密钥等敏感配置 | ✅ 生产环境必须修改 |
| `docker-compose.yml` | 服务编排、资源限制 | 根据硬件调整 |
| `mysql/my.cnf` | MySQL 性能调优 | 根据内存调整 |
| `es/elasticsearch.yml` | ES 配置 | 一般不需要改 |
| `nginx/nginx.conf` | 反向代理、负载均衡 | 根据域名调整 |

## 2. 应用代码类

| 文件 | 作用 | 开发频率 |
|------|------|----------|
| `app/main.py` | 应用入口、中间件配置 | 低 |
| `app/config.py` | 配置读取 | 低 |
| `app/models/*.py` | 数据库模型定义 | 中 |
| `app/routers/*.py` | API 接口定义 | 高 |
| `app/services/*.py` | 业务逻辑实现 | 高 |
| `app/tasks/*.py` | 异步任务 | 中 |

## 3. 数据存储类（容器挂载）

| 目录 | 用途 | 备份建议 |
|------|------|----------|
| `mysql/data/` | 业务数据 | ✅ 每日备份 |
| `es/data/` | 搜索索引 | ✅ 每日备份 |
| `redis/data/` | 缓存数据 | 可选 |
| `grafana/data/` | 监控配置 | 可选 |

## 4. 日志文件类

| 目录 | 内容 | 清理策略 |
|------|------|----------|
| `logs/app/` | 应用运行日志 | 保留 14 天 |
| `logs/nginx/` | 访问/错误日志 | 保留 30 天 |
| `logs/celery/` | 任务执行日志 | 保留 7 天 |


# ============================================================
# 文件大小预估
# ============================================================

初始部署后各目录大小：

```
/data
├── app/                    ~ 5 MB      # 代码
├── mysql/data/             ~ 500 MB    # 初始（随数据增长）
├── es/data/                ~ 1 GB      # 初始（随索引增长）
├── redis/data/             ~ 10 MB     # 缓存
├── grafana/data/           ~ 50 MB     # 监控数据
├── logs/                   ~ 100 MB    # 日志（定期清理）
└── backups/                ~ 0 MB      # 备份目录（自动生成）
```

160GB 磁盘分配建议：
- 系统根分区 `/`：20GB
- 数据分区 `/data`：120GB（本项目使用）
- 日志分区 `/var/log`：10GB
- 交换分区 `swap`：8GB
- 预留：2GB


# ============================================================
# 权限要求
# ============================================================

关键目录权限设置：

```bash
# ES 数据目录必须 777（容器内 elasticsearch 用户需要写入）
chmod 777 /data/es/data

# MySQL 数据目录 755（容器内 mysql 用户）
chmod 755 /data/mysql/data

# 其他目录默认 755 即可
chmod 755 /data/redis/data
chmod 755 /data/rabbitmq/data
```
